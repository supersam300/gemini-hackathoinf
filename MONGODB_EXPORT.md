# MongoDB Atlas Vector Search & Persistence Guide

This guide documents how SimuIDE Web integrates with MongoDB Atlas for circuit persistence and AI-driven semantic search.

## Overview
The application utilizes MongoDB Atlas to:
1. **Persist Projects**: Save and load complete circuit diagrams, including component positions, wiring, and code.
2. **Semantic Search (RAG)**: Store text embeddings generated by Gemini to enable vector-based search across documentation or shared projects.

## Database Architecture

### Collections
- **circuits**: Stores the serialized state of active projects.
- **documents**: Stores text fragments and their corresponding 768-dimensional embeddings for vector search.

### Vector Search Index Configuration
To enable semantic search, a Vector Search index must be created on the `documents` collection:
```json
{
  "fields": [
    {
      "type": "vector",
      "path": "embedding",
      "numDimensions": 768,
      "similarity": "cosine"
    }
  ]
}
```

## Internal Data Models

### Circuit Model (`server/models/Circuit.js`)
Stores the visual and logic state of a simulation project.
```javascript
{
  projectName: String,
  code: String,
  language: String,   // c, cpp, python
  components: Array,  // UI nodes and positions
  connections: Array  // Wire relationships
}
```

### Document Model (`server/models/Document.js`)
Stores data for the AI knowledge base.
```javascript
{
  text: String,
  metadata: Object,
  embedding: [Number] // 768-dim vector
}
```

## API Implementation

- **`POST /api/circuits`**: Serializes the `diagramStore` and `editorStore` from the frontend and saves them to the `circuits` collection.
- **`POST /api/ai/ingest`**: Generates a Gemini embedding for input text and stores it in the `documents` collection.
- **`POST /api/ai/search`**: Converts a natural language query into an embedding and performs a `$vectorSearch` to retrieve relevant context.

## Integration Workflow

1. **Serialization**: The frontend `serializer.ts` utility converts the React Flow canvas state into a MongoDB-friendly JSON format.
2. **Persistence**: The `projectStore.ts` manages the lifecycle of cloud-based saves, tracking the `cloudCircuitId` for updates.
3. **Retrieval**: Upon loading, the application fetches the full document and populates the global Zustand stores, triggering a canvas re-render.
